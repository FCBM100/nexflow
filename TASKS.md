# TASKS — خطة تنفيذ موقع NexFlow
### مرجعية كاملة: `PRD.md`
يُنفَّذ هذا الملف بالترتيب التسلسلي للمراحل. كل مرحلة تحتوي: الهدف، المهام، معيار الإنجاز (Definition of Done).

---

## Phase 0 — البحث والتحضير (Research & Discovery)
**الهدف:** فهم تقنيات الحركة الفعلية في المواقع المرجعية قبل كتابة أي كود.

- [ ] استخدام **Firecrawl** لتفريغ بنية صفحات: `vercel.com`, `linear.app`, `stripe.com`, `framer.com`, `openai.com` — رصد: نوع Scroll Reveal المستخدم، سرعة الانتقالات، طريقة تعامل كل موقع مع المساحات البيضاء.
- [ ] حصر كل الأصول الجاهزة (شعار، بانر، فيديو) والتأكد من فهم تسلسل حركة الفيديو (موجود بالفعل في `PRD.md` §2.3).
- [ ] حسم الأسئلة المفتوحة في `PRD.md` §9 قبل المتابعة لـ Phase 1 (لا تبدأ Hero بدون حسم سؤال "2D vs 3D").

**معيار الإنجاز:** قائمة ملاحظات تقنية مكتوبة من تحليل المواقع المرجعية + إجابات الأسئلة المفتوحة.

---

## Phase 1 — التحضير التقني (Project Setup)
- [ ] `npx create-next-app@latest` مع TypeScript + Tailwind + App Router.
- [ ] تثبيت الحزم: `framer-motion`, `gsap`, `three`, `@react-three/fiber`, `@react-three/drei`, `lenis`.
- [ ] إعداد ESLint + Prettier + تنظيم بنية المجلدات حسب `PRD.md` §7.2.
- [ ] إعداد `next/font` لخطّي العربي/اللاتيني المقترحين (§2.5) مع متغيرات CSS.
- [ ] إعداد Git repository + أول Commit (Initial setup).

**معيار الإنجاز:** مشروع Next.js يعمل محليًا (`npm run dev`) بصفحة فاضية بخط ولون الخلفية الصحيحين.

---

## Phase 2 — نظام التصميم (Design System / Tokens)
- [ ] إضافة ألوان البراند (§2.4) كـ CSS Variables + Tailwind theme extend.
- [ ] بناء Utility classes: `.glass-card` (backdrop-blur + border)، `.neon-glow` (box-shadow/text-shadow بتدرجات السماوي)، `.gradient-text`.
- [ ] مكوّن `Button` أساسي (Primary / Secondary) بحالات: Default / Hover / Focus / Magnetic-ready.
- [ ] مكوّن `GlassCard` أساسي قابل لإعادة الاستخدام بكل الأقسام.
- [ ] توثيق سريع لنظام الـ Spacing/Typography scale المستخدم (Tailwind defaults أو مخصص).

**معيار الإنجاز:** صفحة Storybook-like بسيطة أو قسم تجريبي يعرض كل العناصر الأساسية (ألوان، أزرار، بطاقة) بصريًا متطابقة مع هوية الأصول.

---

## Phase 3 — تنظيم وتحسين الأصول (Asset Pipeline)
- [ ] نقل الملفات وتسميتها حسب البنية المقترحة في `PRD.md` §7.2 (`public/brand/...`).
- [ ] توليد Favicon set كامل (16/32/180/192/512px) من `photo_2026-06-18_20-41-27.jpg`.
- [ ] ضغط/تحسين الفيديو (`logo-animation.mp4`/`.webm`) للويب إذا لزم (التحقق من الحجم الحالي كافٍ أولًا).
- [ ] استخراج Poster Frame ثابت من الفيديو (أول فريم) لاستخدامه كـ `poster` attribute أثناء التحميل.
- [ ] تجهيز `og-banner-1200x630.jpg` ضمن مجلد `public` لاستخدامه مباشرة بالـ Metadata.

**معيار الإنجاز:** كل الأصول بصيغها النهائية موجودة بمجلد `public/brand` وجاهزة للاستدعاء بالكود.

## Phase 4 — الهيكل العام + شاشة التحميل + التنقل
- [ ] `RootLayout`: `<html dir="rtl" lang="ar">`, تحميل الخطوط، تفعيل Lenis Provider.
- [ ] مكوّن `LoadingScreen`: تشغيل فيديو الشعار (مدة معروضة ≤ 2.5 ثانية، Exit Transition بـ GSAP).
- [ ] مكوّن `Navbar`: شفاف → Glass عند التمرير، شعار مصغّر، روابط، زر CTA دائم.
- [ ] التحقق من سلوك Navbar على Mobile (Hamburger menu أو تبسيط الروابط).

**معيار الإنجاز:** الموقع يفتح بشاشة تحميل سينمائية، ثم ينتقل بسلاسة لصفحة فيها Navbar فعّال بالتمرير.

---

## Phase 5 — قسم Hero (الأثقل تقنيًا)
- [ ] بناء العنصر المركزي حسب القرار المحسوم في `PRD.md` §5 (الخيار أ أو ب).
- [ ] مكوّن `NeuralNetworkBackground` (Three.js/R3F أو Canvas2D) — نقاط متصلة تستجيب لحركة الماوس بخفة.
- [ ] مكوّن `CursorGlow` (Desktop only، يُعطَّل تلقائيًا على Touch devices).
- [ ] مكوّن `MagneticButton` wrapper لزري الـ Hero (أساسي/ثانوي).
- [ ] كتابة نص العنوان والنص الفرعي والأزرار (نص ثابت من `PRD.md` §4.3 — بدون تعديل).
- [ ] Entry animation للعنوان (Stagger بالكلمات أو الأحرف عبر GSAP/Framer Motion).
- [ ] اختبار أداء هذا القسم خصوصًا على جهاز متوسط المواصفات (هذا القسم الأكثر عرضة لمشاكل الأداء).

**معيار الإنجاز:** Hero يعمل بسلاسة 60fps على جهاز متوسط، والنصوص والأزرار تعمل بكل التفاعلات المطلوبة.

- [ ] قسم الحل بتصميم بصري متمايز (خلفية/توهج مختلف عن البطاقات).
- [ ] كتابة المحتوى الثابت من `PRD.md` §4.4 بدون تعديل.

**معيار الإنجاز:** القسم يظهر بانتقال مختلف عن Hero، والبطاقات تستجيب بصريًا لحركة الماوس.

**معيار الإنجاز:** القسم يبدو "حدث استثنائي" بصريًا مقارنة ببقية الصفحة، تمامًا كما طلب الـ PRD.

## Phase 9 — قسم الإحصائيات (Animated Counters)
- [ ] مكوّن `AnimatedCounter` بـ Framer Motion `useInView` + `animate` (يبدأ العدّ عند الدخول للشاشة فقط، لا عند تحميل الصفحة).
- [ ] حسم شكل البطاقتين غير الرقميتين حسب إجابة السؤال المفتوح #1 في `PRD.md`.
- [ ] تنسيق الأرقام (إنكليزية أو عربية هندية) حسب القرار في `PRD.md` §7.3.

**معيار الإنجاز:** الأرقام تُعدّ بصريًا بسلاسة عند ظهور القسم، مرة واحدة فقط (لا تتكرر عند كل scroll في/خارج المنطقة).

## Phase 12 — الفوتر
- [ ] روابط سريعة + أيقونات التواصل (Telegram/WhatsApp/Email).
- [ ] حقوق النشر بالسنة الحالية (ديناميكي عبر `new Date().getFullYear()`).
- [ ] (اختياري) شعار الأنيميشن مصغّر وشبه شفاف كخلفية زخرفية.

**معيار الإنجاز:** الفوتر مكتمل وكل الروابط تعمل (لا روابط placeholder منسية).

---

## Phase 13 — الصقل العام للتفاعلات (Global Interactions Polish)
- [ ] مراجعة شاملة: كل قسم يجب أن يملك Reveal Effect **مختلف** عن الذي قبله (لا تكرار نمط واحد بكل الصفحة).
- [ ] تفعيل Parallax بين الأقسام الانتقالية (GSAP `scrub`) بدرجة خفيفة غير مزعجة.
- [ ] اختبار `prefers-reduced-motion` فعليًا (تفعيله من نظام التشغيل والتأكد من تعطّل الحركات الثقيلة).
- [ ] مراجعة اتجاه كل حركة/Parallax بسياق RTL (راجع `PRD.md` §7.3) — تصحيح أي اتجاه يبدو "معكوسًا" بصريًا.

**معيار الإنجاز:** تجربة تمرير الصفحة كاملة من البداية للنهاية تبدو متناغمة ومتنوعة، لا رتيبة ولا فيها أي حركة بـ"اتجاه خاطئ" بصريًا.

## Phase 14 — تحسين الأداء (Performance)
- [ ] تحميل Three.js/R3F بـ `dynamic import` (lazy, ssr: false).
- [ ] فحص Lighthouse (Mobile + Desktop) ومطابقته لأهداف `PRD.md` §8.
- [ ] تحسين الصور (`next/image`) والفيديو (lazy load بعد Hero مباشرة فقط، لا قبل).
- [ ] فحص حجم الـ Bundle النهائي (`next build` + تحليل الـ output) وتقليل أي مكتبة زائدة.

**معيار الإنجاز:** نتيجة Lighthouse Performance ≥ 90 على Mobile، LCP < 2.5s.

## Phase 16 — الاختبار والجودة (QA)
- [ ] استخدام **Playwright** لأخذ Screenshots تلقائية على مقاسات: `375px` (Mobile)، `768px` (Tablet)، `1440px` (Laptop)، `1920px` (Desktop) لكل قسم — مراجعة بصرية لكل لقطة.
- [ ] فحص RTL فعليًا على كل المقاسات (لا انكسار اتجاه، لا تداخل نصوص).

- [ ] فحص الوصولية (Keyboard navigation + Focus states + تباين النص).

**معيار الإنجاز:** لا أخطاء بصرية أو وظيفية على أي من المقاسات/المتصفحات المختبرة.
