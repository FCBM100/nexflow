# PRD — موقع NexFlow الرسمي
### Product Requirements Document
**النوع:** Portfolio + Business Landing Page
**الفئة التصميمية:** Premium SaaS / Futuristic AI
**اللغة والاتجاه:** عربي — RTL (أساسي)
**الإصدار:** 1.0 — جاهز للتطوير

---

## 1. نظرة عامة على المشروع

### 1.1 ما هو NexFlow؟
موقع تعريفي واحترافي (Portfolio + Landing Page) لخدمات تقنية تحت براند **NexFlow**، يقدّم:
- تطوير مواقع وتطبيقات ويب.
- بناء بوتات ذكية لواتساب وتليغرام (خدمة عملاء، حجوزات).
- أتمتة العمليات التشغيلية للأعمال الصغيرة والمتوسطة.

شعار البراند: **"Automate. Integrate. Elevate."**

### 1.2 المشكلة التي يحلها الموقع
أصحاب البيجات والمحلات والمشاريع الصغيرة يفقدون عملاء بسبب:
1. التأخر في الرد على الاستفسارات المتكررة.
2. غياب هوية رقمية احترافية (لا موقع يعكس مستوى العمل).
3. إدارة يدوية مكررة للحجوزات والطلبات تستهلك الوقت.

### 1.3 هدف الموقع (Business Goal)
تحويل الزائر إلى **عميل محتمل (Lead)** يضغط على زر التواصل عبر تليغرام، بعد أن:
- يثق بالمستوى التقني العالي للبراند (من خلال جودة التصميم نفسه — الموقع هو أكبر دليل على الخدمة المباعة).
- يفهم الخدمات المعروضة بوضوح.
- يرى دليل نجاح ملموس (Case Study بأرقام حقيقية).

### 1.4 الجمهور المستهدف
| الفئة | الحاجة |
|---|---|
| أصحاب البيجات (Instagram/Facebook Pages) | موقع احترافي + بوت رد تلقائي |
| أصحاب المحلات التجارية | نظام حجز وطلبات مؤتمت |
| الشركات الناشئة الصغيرة والمتوسطة | حلول أتمتة شاملة وحلول AI للأعمال |

### 2.3 فيديو الشعار المتحرك — Animated Logo Loop
**الملفات:** `logo-animation.mp4` + `logo-animation.webm` (نسخ نهائية مزدوجة الصيغة للأداء الأفضل على الويب) — المصدر الأصلي: `video_2026-06-18_20-42-07.mp4`.

**المواصفات التقنية:** 960×960px — مدة **6.04 ثانية** — 24fps — خلفية شفافة/داكنة قابلة للتكرار (Seamless Loop).

**تحليل تسلسل الحركة (مهم جدًا للتنفيذ):**
1. **(0:00)** الشعار الثابت بوضعه الطبيعي — الروبوت في المركز والأيقونات الخمس على الدائرة.
2. **(0:01–0:03)** الحلقة الخارجية "تتفجّر" إلى جزيئات وشظايا متوهجة (Glow Particles + Motion Blur Trails) تدور حول الروبوت بسرعة، كأنها نقل بيانات أو نبضة طاقة (Energy Pulse).
3. **(0:04–0:06)** الجزيئات تتجمّع وتُعيد تشكيل حلقة مضاعفة (Double Ring) أكثر توهجًا وسماكة حول الروبوت، ثم تهبط الإضاءة تدريجيًا لتعود لنقطة البداية → **Loop سلس بدون قفلة مرئية.**

**الاستخدام المقترح (3 استخدامات فعلية):**
| الموضع | الغاية |
|---|---|
| **شاشة التحميل (Loading Screen)** | تشغيل اللوب مرة واحدة (أو دورة ونصف) أثناء تجهيز الموقع — هذا هو "بناء شبكة الذكاء الاصطناعي" المطلوب في الطلب الأصلي، وهو **جاهز فعليًا ولا يحتاج إعادة بناء بـ Three.js** |
| **خلفية Hero Section (Ambient Loop)** | تشغيله بصيغة WebM شفافة بحجم صغير في الزاوية أو خلف العنوان، بحلقة تكرار لا نهائية وبصوت معطّل (muted, autoplay, loop, playsinline) |
| **Footer / About** | عنصر زخرفي صغير بحجم مصغّر يعكس استمرارية البراند |

### 2.4 لوحة الألوان — Color Palette (مؤكَّدة ومطابقة للأصول)
```
--color-primary:    #00E5FF   /* السماوي الأساسي — العيون، الإضاءة، الخطوط */
--color-secondary:  #0099FF   /* الأزرق المتوسط — التدرجات */
--color-accent:     #66F5FF   /* السماوي الفاتح — Highlights */
--color-background: #050B18   /* الخلفية الأساسية الداكنة */
--color-surface:    #0D172A   /* خلفية البطاقات (Glassmorphism base) */
```
ملاحظة تقنية: نسبة التباين بين النص الرمادي الفاتح والخلفية `#050B18` يجب أن تُختبر فعليًا لتحقيق WCAG AA (4.5:1 كحد أدنى) — اللون النيون الزاهي مناسب للعناوين، لكن النصوص الطويلة (Body Text) تحتاج رمادي فاتح مائل للأبيض (`#B8C4D9` تقريبًا) وليس السماوي الكامل لتفادي إجهاد القراءة.

### 2.5 الخط — Typography (مقترح، قابل للتبديل بسهولة عبر `next/font`)
بما أن الإلهام التصميمي مذكور صريحًا (Vercel, Linear, Stripe, Framer, OpenAI)، الاقتراح:
- **العربي:** IBM Plex Sans Arabic (أو Almarai كبديل) — خط هندسي نظيف يتماشى مع طابع SaaS.
- **اللاتيني/الأرقام:** Geist (خط Vercel نفسه، مفتوح المصدر، متاح عبر next/font/google أو محليًا) أو Inter كبديل.
- لا تُثبَّت هذه الخيارات كقرار نهائي صارم — أي خط بنفس الروح الهندسية البسيطة يفي بالغرض.

## 3. مبادئ التصميم (Design Direction)

| المبدأ | يعني عمليًا |
|---|---|
| Premium SaaS Minimal | مساحات بيضاء (سوداء هنا) كثيفة، لا ازدحام، عنصر واحد بكل مرة |
| Futuristic AI | توهج نيون محسوب، لا ألوان زائدة عن البالتة المحددة |
| Glassmorphism محسوب | `backdrop-blur` + حدود شبه شفافة — يُستخدم في البطاقات فقط، لا في كل العناصر |
| الحركة بالمعنى لا بالمبالغة | كل أنيميشن يخدم فهم المحتوى (Reveal يدل على تسلسل القراءة)، وليس حركة لمجرد الحركة |
| السرعة فوق كل شيء | لو التصميم خرافي بصريًا لكن بطيء التحميل = فشل المشروع كاملًا |

**مرجعيات مباشرة:** vercel.com / linear.app / stripe.com / framer.com / openai.com — يُنصح بتحليل تقني فعلي لها (موجود في TASKS.md، مرحلة البحث).

## 4. خريطة الموقع وتفصيل الأقسام

### 4.1 شاشة التحميل (Loading Screen)
- تشغيل فيديو/أنيميشن الشعار (القسم 2.3) كعنصر مركزي وحيد على خلفية `#050B18` صافية.
- شريط تقدّم بسيط أو نسبة % متحركة (اختياري) أسفل الشعار.
- Transition خروج: Mask/Wipe أو Fade مع تكبير خفيف (Scale) للشعار قبل اختفائه.
- **المدة القصوى المسموحة: 2.5 ثانية** (حتى لو الفيديو 6 ثوان، تُعرض دورة جزئية فقط — لا تجعل الزائر ينتظر أنيميشن كامل).

### 4.2 شريط التنقل (Navbar)
- شفاف فوق المحتوى، يتحول لـ Glass عند التمرير (`backdrop-blur` + خلفية `surface` بشفافية).
- الشعار المصغّر (يسار في RTL = الجهة البصرية اليمنى فعليًا، انتبه لمنطق RTL).
- روابط: الخدمات، الأعمال، خطوات العمل، تواصل.
- زر CTA مصغّر دائم الظهور: "تواصل عبر تليغرام".

### 4.3 Hero Section
**المحتوى (نص ثابت، لا تغيير):**
- العنوان: **"تفرّغ لتطوير مشروعك.. واترك إدارة بيجك وأعمالك للأنظمة الذكية!"**
- النص الفرعي: *"أنا شريكك التقني. أساعد أصحاب البيجات، المحلات، والشركات الناشئة في بناء مواقع إلكترونية سريعة واحترافية، وتصميم بوتات ذكية تُدير حجزك وترد على زبائنك، مع أتمتة كاملة لأعمالك لتعمل على مدار الساعة بدون مجهود يدوي."*
- زر أساسي: 🚀 ابدأ بأتمتة عملك وبناء موقعك الآن
- زر ثانوي: 📂 شاهد أعمالي

**العناصر البصرية:**
- مركز الـ Hero: عنصر AI مركزي (انظر القسم 5 — قرار تصميمي مهم بخصوص 2D vs 3D).
- خلفية Neural Network متحركة (نقاط متصلة بخطوط، تتفاعل خفيفًا مع حركة الماوس).
- Cursor Glow يتبع الماوس (دائرة توهج سماوي ناعمة خلف المؤشر).
- الأزرار: Magnetic Effect (تنجذب البطاقة/الزر نحو المؤشر عند الاقتراب).

### 4.4 قسم المشكلة والحل
**العنوان:** هل تعاني من هذه المشاكل في مشروعك؟

**3 بطاقات (Glass Cards بتأثير 3D Tilt عند الـ hover):**
1. **خسارة الزبائن** — بسبب التأخر في الرد على الرسائل والاستفسارات المكررة.
2. **غياب الهوية الرقمية** — عدم امتلاك موقع خاص يعرض عملك بشكل احترافي ويثبت وجودك بالسوق.
3. **المجهود اليدوي المكرر** — ضياع وقتك يوميًا في تسجيل الحجوزات ومتابعة الطلبات يدويًا.

**قسم الحل (تصميم متمايز — ربما خلفية بتدرج أو توهج مختلف):**
💡 *"الحل عندي: أنا هنا لأكون شريكك التقني؛ أبني لك موقعًا يُمثّل براندك، وأربطه بأنظمة أتمتة وبوتات ذكية تدير عملياتك وتوفر وقتك وجهدك."*

### 4.5 قسم الخدمات
**العنوان:** كيف أساعد مشروعك على النمو؟

**3 بطاقات مع أيقونات متحركة (SVG animated icons أو Lottie خفيف):**
1. 🌐 **تطوير المواقع الإلكترونية الاحترافية** — تصميم وبناء مواقع وتطبيقات ويب سريعة، حديثة، ومتجاوبة بالكامل مع الهواتف.
2. 🤖 **بوتات خدمة العملاء والحجز التلقائي** — بوتات ذكية لواتساب وتليغرام ترد فورًا وتدير الحجوزات.
3. ⚙️ **أتمتة العمليات بالكامل** — ربط جميع الأنظمة معًا لتعمل تلقائيًا بدون تدخل يدوي.

### 4.6 قسم قصة النجاح (Case Study)
**يجب أن يكون تصميمه مختلفًا بصريًا عن باقي الأقسام** (مثلًا: تخطيط Split-screen، أو خلفية Surface غامقة بحدود متوهجة، أو محاكاة واجهة شاشة هاتف/بوت حقيقية).

**العنوان:** ⚙️ نظام مبيعات وحجز مؤتمت بالكامل

**الوصف:** *"قمت بتطوير نظام ذكي متكامل لصالح محل تجاري؛ يشمل بوت لخدمة العملاء وحجز الطلبات. البوت يقوم بالرد الفوري، يتيح للزبائن تصفح المنتجات المتوفرة في المحل لحظة بلحظة، ومن ثم إتمام الحجز تلقائيًا وتحديث البيانات مباشرة."*

**النتائج (تُعرض كنقاط + تتصل بعدادات القسم 4.7):**
- تقليل وقت الرد بشكل كبير.
- تنظيم كامل للحجوزات.
- تقليل الأخطاء البشرية.
- تجربة أفضل للزبائن.

### 4.7 قسم الإحصائيات (Animated Counters)
| القيمة | الوصف |
|---|---|
| 24/7 | تشغيل مستمر |
| 100% | أتمتة للمهام المتكررة |
| ثوانٍ معدودة | للرد على العملاء |
| — | زيادة الإنتاجية بشكل ملحوظ |

> ⚠️ **ملاحظة تحتاج قرارك** — موجودة أيضًا في قسم الأسئلة المفتوحة (9): إحصائيتان فقط (`24/7`, `100%`) رقميتان بشكل صريح يصلح لها تأثير Count-Up حقيقي. "ثوانٍ معدودة" و"زيادة الإنتاجية بشكل ملحوظ" نصوص وصفية بلا رقم ثابت. التنفيذ سيُعاملهما كبطاقة (أيقونة + نص) بدل عداد متحرك، إلا إذا زوّدتني برقم تقريبي حقيقي (مثلًا: "أقل من 10 ثوانٍ" أو "زيادة 40% بالإنتاجية").

### 4.8 قسم خطوات العمل (Process Timeline)
خط زمني تفاعلي أفقي (Desktop) / عمودي (Mobile)، يتفعّل كل عنصر عند دخوله الشاشة (Scroll-triggered):
1. تحليل المشروع
2. تصميم الحل
3. التطوير والبرمجة
4. الاختبار والتحسين
5. الإطلاق والمتابعة

### 4.9 قسم التواصل النهائي (Final CTA)
خلفية مبهرة بتأثيرات ضوئية (Glow + Gradient Mesh متحرك بهدوء).

**العنوان:** جاهز لتبني موقعك وتنقل مشروعك للمستوى القادم؟

**النص:** *"لا تخسر عميلًا آخر بسبب التأخر في الرد، واجعل لعملك وجودًا رقميًا حقيقيًا بموقع احترافي. اضغط على الزر أدناه ودعنا نتحدث على التليغرام لنبدأ بالتخطيط لمشروعك وتسهيل شغلك!"*

**زر ضخم متوهج:** 💬 تواصل معي عبر تليغرام فورًا (Magnetic + Pulse Glow مستمر لجذب الانتباه)

## 5. قرار تصميمي مهم: عنصر الروبوت في Hero (2D Asset vs 3D Model)

الطلب الأصلي يذكر "نموذج روبوت AI ثلاثي الأبعاد"، لكن الأصول الجاهزة الحالية (الشعار، البانر، الفيديو) كلها **رسوم Flat/Vector ثنائية الأبعاد**، وليست موديل 3D. هذا قرار يحتاج اختيارك بين خيارين:

**الخيار أ — الأسرع والمتماشي مع الهوية الحالية (موصى به):**
استخدام الشعار المتحرك الجاهز (القسم 2.3) كمركز الـ Hero، مع طبقة `Three.js`/`R3F` خلفه تنتج حقل جزيئات (Particle Field) وخطوط شبكة عصبية تستجيب لحركة الماوس — هذا يحقق "الإحساس بالـ 3D والعمق" بدون الحاجة لنمذجة روبوت كامل من الصفر، ويحافظ 100% على الهوية البصرية المعتمدة فعليًا في كل أصول البراند.

**الخيار ب — الأقوى بصريًا لكن الأبطأ والأعلى تكلفة تنفيذ:**
بناء/طلب موديل 3D فعلي مستوحى من تصميم الروبوت بنفس الملامح (الرأس الدائري، العينان السماويتان، الهوائي)، وعرضه بـ React Three Fiber مع إضاءة وحركة دوران بطيئة. يتطلب وقت تطوير أطول واختبار أداء أدق على الأجهزة الضعيفة.

> الافتراض الافتراضي في TASKS.md هو **الخيار أ**، إلا إذا أكدت رغبتك بالخيار ب صريحًا قبل بدء مرحلة Hero.

| Cursor Glow | Canvas/CSS + JS مخصص | الصفحة كاملة (Desktop فقط) |
| Magnetic Buttons | Framer Motion (`useMotionValue`) | كل أزرار CTA |
| 3D Card Tilt | Framer Motion / vanilla `transform: perspective` | بطاقات المشكلة والخدمات |
| Neural Network Background | Three.js / R3F أو Canvas2D خفيف | Hero (و اختياريًا الخلفية العامة بشفافية منخفضة) |
| Floating Particles | Three.js particles أو CSS-only للأقسام الأخف | Hero + Final CTA |
| Parallax متعدد الطبقات | GSAP ScrollTrigger (`scrub`) | بين الأقسام الانتقالية |
| Animated Counters | Framer Motion `useInView` + `animate` | قسم الإحصائيات |
| Loading Screen Build | فيديو الشعار الجاهز + GSAP لتنسيق الخروج | شاشة التحميل فقط |

**قاعدة أساسية:** كل حركة تحترم `prefers-reduced-motion` — تُعطَّل الحركات الثقيلة (Particles/Parallax) تلقائيًا للمستخدمين الذين يفعّلون هذا الخيار في نظامهم، مع الحفاظ على الانتقالات الأساسية (Fade) فقط.

## 7. البنية التقنية (Technical Architecture)

### 7.1 الـ Stack المعتمد (كما حدّدته صريحًا)
```
Next.js 15 (App Router) + TypeScript
Tailwind CSS
Framer Motion        → تفاعلات UI وانتقالات المكوّنات
GSAP + ScrollTrigger  → Scroll Reveal و Parallax المعقّد
Three.js + React Three Fiber → الخلفية ثلاثية الأبعاد بالـ Hero
Lenis                 → Smooth Scroll
```

### 7.2 بنية المجلدات المقترحة
```
src/
 ├─ app/
 │   ├─ layout.tsx        (RTL + Fonts + Lenis Provider)
 │   └─ page.tsx
 ├─ components/

 │   ├─ ui/               (Button, GlassCard, MagneticWrapper, CursorGlow)
 │   └─ three/            (ParticleField, NeuralNetworkScene)
 ├─ lib/                  (animations helpers, hooks)
 └─ public/
     └─ brand/
         ├─ nexflow-icon.png        (من photo_2026-06-18_20-41-27.jpg)
         ├─ og-banner.jpg           (من og-banner-1200x630.jpg)
         ├─ logo-animation.mp4
         └─ logo-animation.webm
```

### 7.3 استراتيجية RTL
- `<html dir="rtl" lang="ar">` على مستوى الـ Layout.
- استخدام Tailwind Logical Properties (`ms-`, `me-`, `ps-`, `pe-`) بدل `ml-`/`mr-` لضمان انعكاس صحيح تلقائيًا.
- مراجعة اتجاه حركات GSAP/Parallax يدويًا (المحتوى في RTL "يدخل" من اليسار بصريًا غالبًا، لا من اليمين كما في LTR) — تحديد الاتجاه الصحيح لكل Reveal بالتجربة الفعلية لا بالافتراض.
- الأرقام في العدادات تُعرض بأرقام إنكليزية (1, 2, 3) أو عربية هندية (١,٢,٣) — قرار تصميمي صغير يُفضَّل تثبيته من البداية لتناسق الموقع (الافتراض: أرقام إنكليزية لتماشيها مع الطابع التقني SaaS، قابل للتغيير).

### 7.4 SEO والمشاركة الاجتماعية
- `og:image` و `twitter:image` → `og-banner-1200x630.jpg` (جاهز فعليًا بالمقاس الصحيح).
- Title/Description بالعربية مع دعم `og:locale = ar_AR`.
- Favicon set كامل (16x16 إلى 512x512 + Apple touch icon) من ملف الأيقونة `photo_2026-06-18_20-41-27.jpg`.
- `sitemap.xml` و `robots.txt` (موقع صفحة واحدة، بسيط لكن لا يُهمَل).

## 9. أسئلة مفتوحة تحتاج قرارك قبل التنفيذ

1. **الإحصائيات غير الرقمية** (القسم 4.7) — هل تريد أرقامًا تقريبية حقيقية لـ "ثوانٍ معدودة" و"زيادة الإنتاجية"، أم تبقى بطاقات نصية بدون عداد متحرك؟
2. **عنصر Hero المركزي** (القسم 5) — الخيار أ (شعار 2D + جزيئات Three.js) أم الخيار ب (موديل 3D جديد كليًا)؟
3. **رابط زر التليغرام** — هل لديك يوزر/رابط تليغرام نهائي جاهز لربطه بالأزرار الآن، أم placeholder مؤقت؟
4. **نسخة إنكليزية** — هل الموقع عربي RTL فقط (كما هو محدد حاليًا)، أم هناك خطة لاحقة لتبديل لغة EN/AR؟
5. **الدومين والاستضافة** — هل الدومين محجوز فعليًا؟ هذا يؤثر على ترتيب مرحلة النشر بـ TASKS.md.
